JavaScript 시작 (3-8) - 동적 타이핑

2022/05/02 카테고리 JavsScript

1. 정적 타입

모든 값은 데이터 타입을 갖는데 변수는 데이터 타입을 가질까?
C나 자바 같은 정적 타입 언어는 변수를 선언 할 때 변수의 할당 할 수 있는 값의 종류를 사전에 선언해야하는데 이를 명시적 타입 선언이라고 한다.

2. 동적 타입

자바스크립트는 동적 타입 언어로 정적 타입 언어와 다르게 변수를 선언할 때 타입을 선언하지 않으며, 할당되는 값의 타입에 따라 결정된다.

정적 타입 언어 동적 타입 언어
변수 선언 시점에 변수 타입이 결정되고 변수의 타입을 변경할 수 없다. 값을 할당하는 시점에 변수의 타입이 동적으로 결정되고 변수의 타입을 언제든지 자유롭게 변결할 수 있다.
C, C++, JAVA, Kotlin, Go, Haskell, Rust, Scala 등 Javascript, Python, PHP, Ruby, Lisp, Perl 등

값에 따라 타입 변경 예

기본적으로 변수는 타입을 갖지 않는다. 다만 값이 타입을 갖고 그에 의해 변수의 타입이 결정된다.


동적 타입 언어의 주의점

변수의 타입이 값에 따라 언제든지 변경될 수 있어 확인 이전에는 타입을 확신할 수 없다. 결국 동적 타입 언어는 유연성은 높지만 신뢰성은 떨어진다.

  1. 변수의 개수가 많으면 많을수록 오류가 발생할 확률도 높아지며, 변수의 무분별한 남발은 금물이며, 필요한 만큼 최소한으로 유지
  2. 변수의 유효 범위가 넓으면 넓을수록 변수로 인해 오류가 발생할 경우 오류의 원인을 특정하기 어렵다.
  3. 전역 변수를 최대한 사용하지 않도록 한다. 전역 변수는 의도치 않게 값이 변경될 가능성이 높고 다른 코드에 영향을 줄 가능성도 높으므로 프로그램의 복잡성을 증가시키고 처리 흐름을 추적하기 어렵게 만들고, 오류가 발생할 경우 오류의 원인을 특정하기 어렵게 만든다.
  4. 변수보다는 상수를 사용해 값의 변경을 억제한다. (const 명렁어)
  5. 변수 이름은 변수의 목적이나 의미를 파악할 수 있도록 네이밍한다. 이는 협업과 생산성 향상에 도움을 준다.
