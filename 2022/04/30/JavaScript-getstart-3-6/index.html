JavaScript 시작 (3-6) - 원시 타입의 원시 값

2022/04/30 카테고리 JavsScript

자바스크립트의 데이터 타입은 크게 원시 타입과 객체 타입으로 분류된다. 자바스크립트는 객체 기반의 언어이며, 자바스크립트를 이루고 있는 거의 모든 것이 객체다.
숫자, 문자열, boolean, undefined, null, symbol 타입을 제외한 값은 모두 객체 타입(object type)이다.

원시 타입 객체 타입
원시 타입의 값은 변경 불가능한 값이다. 객체 타입의 값은 변경 가능한 값이다.
값을 변수에 할당하면 실제 값이 저장된다. 변수에 할당하면 참조 값이 저장된다.
변수를 다른 변수에 할당하면 원본의 값이 복사되어 전달된다. (값에 의한 전달) 변수를 다른 변수에 할당하면 원본의 참조 값이 복사되어 전달된다.(참조에 의한 전달)

원시 값

1. 변경 불가능한 값

원시값은 변경 불가능한 값으로 한번 생성된 원시 값은 읽기 전용 값으로 변경할 수 없다.

원시 값이 변경이 불가능한 것이지 변수의 할당된 값이 변경이 불가능하다는 것은 아니다.


위 예제처럼 메모리에 저장된 값이 바뀌는 것이 아니라 변수가 할당된 값의 메모리 주소로 연결되는 것이므로 원시 값이 변경 된것이 아니라 단지 변수가 해당 값의 메모리 주소를 찾아 갔다고 보는 것이 맞다.

2. 문자열과 불변성

문자열은 유사 배열 객체이면서 이터러블이므로 배열과 유사하게 각 문자에 접근할 수 있다.


유사배열 객체란?

마치 배열처럼 인덱스로 프로퍼티 값에 접근할 수 있고 length 프로퍼티를 갖는 객체를 말하며, for 문으로 순회할 수도 있다.


이미 생성된 문자열의 일부 문자를 string[0] = “H” 처럼 변경을 하더라도 반영이 되지는 않는다.

변수에 새로운 문자열을 재할당하는 것은 가능하며, 이는 위에 나온 설명처럼 변수가 해당 메모리 주소를 찾아가는 것이다.


3. 값에 의한 전달

위 내용에서 보면 변수 copyNumber에 변수 number을 할당하였고 후에 number의 값을 변경하였음에도 마지막 줄의 결과 값은 변경되지 않았다.
이는 변수 number의 값이 변수 copyNumber의 값으로 복사되어 할당 되었기 때문이다.

두 변수의 원시 값은 서로 다른 메모리 공간에 저장된 별개의 값이 되어 어느 한쪽에서 재할당을 통해 변경하더라도 서로 간섭할 수 없다.
